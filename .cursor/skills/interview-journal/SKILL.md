# Interview Journal（面试记录沉淀）

## 目标

把每次面试的题目与复盘，**结构化追加**到本项目的 `Flutter面试题汇总.md`，方便下次面试前快速回看。

## 适用触发

当用户说到以下意图时启用：

- 面试记录 / 面试复盘 / 面试回顾
- 这次面试问了什么 / 把题目整理到 md
- 根据 JD 做对照复习点、补充准备清单

## 写入位置与规则

- **主文件**：`Flutter面试题汇总.md`
- **追加位置**：优先追加到 `## 7. 面试问答速记（Q&A）` 章节末尾；如果内容更像“整场面试复盘”，则在该章节下新增日期小节。
- **保持短小**：每个问答条目优先写成「一句话结论 + 3~5 个要点」。
- **去重合并**：如果与现有条目重复，优先合并到对应小节，补充“易错点/面试追问”。
- **标注关键词**：每个条目末尾附上 3~8 个关键词（便于 Ctrl+F）。

## 输出模板（追加到 md 的格式）

在 `## 7. 面试问答速记（Q&A）` 下追加：

```markdown
### 7.x [题目标题]

- **一句话结论**：...
- **要点**：
  - ...
  - ...
  - ...
- **常见追问**：...
- **关键词**：`...` `...` `...`
```

## 整场面试复盘模板（可选）

当用户提供完整面试信息（公司/岗位/轮次/题目列表）时，用以下结构追加：

```markdown
### YYYY-MM-DD｜公司｜岗位｜第 N 轮（形式：AI/电话/视频/现场）

- **整体反馈**：...
- **题目清单**：
  1. ...
  2. ...
- **不会/答得不好的点**：
  - ...
- **补强计划（按优先级）**：
  - P0：...
  - P1：...
- **关键词**：`...` `...`
```

## 追加流程

1. 读取 `Flutter面试题汇总.md`，定位第 7 章（或创建该章）。
2. 将用户这次提出的新问题/复盘内容结构化成条目。
3. 去重并合并到最相关的小节；必要时新增 `7.x`。
4. 写回 md，保持目录编号/标题一致。
